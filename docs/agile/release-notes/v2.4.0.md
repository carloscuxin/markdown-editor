## Highlights

Instant Payments is now **generally available** over the partner-bank API. Customers can move funds between Helios-connected accounts with settlement in under 10 seconds, 24/7.

## New features

- **Instant payments API** (`POST /v2/payments`) with idempotency support — [HEL-142](../stories/hel-142-send-instant-payment.md).
- **Real-time status webhooks** for payment lifecycle events.
- **Configurable per-account limits** in the web console.

## Improvements

- Ledger reservation call is now synchronous, cutting settlement latency by ~40%.
- Clearer API error codes (`INSUFFICIENT_FUNDS`, `PAYMENT_HELD`).

## Fixes

- Resolved a rounding discrepancy in multi-currency balance display.
- Fixed webhook delivery ordering under high load.

## Known issues

- Web console tracking view (HEL-144) ships in v2.4.1; use the API status endpoint until then.

## Upgrade notes

No breaking changes. The `/v1/transfers` endpoint remains available but is **deprecated** — migrate to `/v2/payments` before v3.0.

## Traceability

Delivered stories: HEL-142. Screened by fraud scoring pilot from [EPIC-002](../epics/epic-002-fraud-risk-screening.md).
