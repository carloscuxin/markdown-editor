# User Stories

A **user story** is a small, independently valuable slice of functionality that can be delivered within a sprint. Stories roll up to [epics](../epics/index.md).

## Story format

> **As a** \<role\>, **I want** \<capability\>, **so that** \<benefit\>.

Each story carries acceptance criteria in **Given / When / Then** form and a clear Definition of Done.
